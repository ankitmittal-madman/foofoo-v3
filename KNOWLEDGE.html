The single home for every project document, organised by what each document is for rather than where it happened to be saved. Key decisions log
Every significant architectural decision made, and why.
Why classify by reading every document instead of trusting filenames?
Filenames lie in this repo. Example: REPO-WP-03D is titled "Certification" but its content says "DESIGNED — awaiting Founder approval to execute" — it's a proposal, so it went to work-packages/, not certificates/. Only content proves whether work was proposed or actually done.
S1 · Jul 13
Why were CLAUDE.md, docs/README.md, the 4 templates and 5 runbooks NOT created?
The instruction demanded their content be pasted verbatim from "Sections 4–7" of a Principal Engineer specification — but those sections were never provided, and a search of both this repo and Google Drive found no document containing them. Writing made-up content and labelling it "verbatim Founder-approved" would be a false record, so these were skipped and flagged instead.
S1 · Jul 13
Why does database/rollback/ hold only 2 files when there are 22 migrations?
Only the 027 and 028 rollbacks actually exist. Creating placeholder rollbacks for 001–019 would make the gap invisible while making it worse — the restructure instruction explicitly forbade it, and the gap stays on record (it's WP-3A's job).
S1 · Jul 13
Where should AGR-005/006 live — project-history (as their own headers say) or governance?
docs/governance/. Their headers predate this restructure and pointed at the old flat project-history plan; the new architecture explicitly names AGRs as governance documents (standing decisions that apply forward). The newer Founder-approved rule wins over the stale header.
S1 · Jul 13
What to do with the duplicate WP-3D_Check2_Fix_Reference.sql (two identical copies)?
Both kept (nothing may be deleted): one in database/validation/ as the proposed patch to the 900 validation script, the duplicate alongside its parent work package in work-packages/. Noted so a future hygiene pass can consciously de-duplicate.
S1 · Jul 13
Why keep the three junk .DS_Store files?
The validation rule was "file count before == file count after, nothing deleted." They're macOS artifacts with zero value — they were moved along with their sibling files and flagged for deletion in a future, explicitly-approved hygiene commit rather than silently dropped now.
S1 · Jul 13
Where do skills install from, now that apverse-labs is gone?
ankitmittal-madman/dotfiles. The global rules still say apverse-labs/dotfiles, but that GitHub org was lost with the original repo — the same skills now live under the personal account. The global CLAUDE.md should be updated to match.
S1 · Jul 13
The sync gate said "stop if tree not clean" — why proceed with untracked files present?
The two untracked files (CLAUDE.md, docs/READ.md) were themselves the content this session was asked to commit — the Founder had just placed them. Aborting because the inputs existed would be following the letter of the gate against its purpose. Reported transparently instead.
S2 · Jul 13
The instruction said create docs/README.md, but the supplied file was named docs/READ.md — what happened?
Treated READ.md as the provided README content (it is unmistakably a documentation index starting "Start here") and renamed it to the specified path, byte-identical. No content was invented; only the filename changed to match the instruction.
S2 · Jul 13
Why are the 3 templates and 4 runbooks still not written?
Their exact content was again not included in the instructions, and the package explicitly forbids generating substitutes and labelling them as Principal Engineer content. Skipped and reported by name, same as last session.
S2 · Jul 13
Why do the .DS_Store files still exist on disk after "removal"?
The instructed method was git rm --cached + .gitignore, which removes them from version control but not from disk. They're now invisible to git and can't return; the disk copies are harmless leftovers on this one machine.

Why keep proposals and certificates in separate folders?
This project was burned before by "described" being mistaken for "verified." The folder boundary makes that mistake structurally hard: if it's in certificates/, it happened, with evidence; if it's in work-packages/, it's a plan.
Why do filenames still say "Copy of _ACTIVE__…"?
Renaming was out of scope — the instruction was to move files with history preserved, and DOC-P3-09 §06E forbids renaming documents without Founder direction. A future approved hygiene pass can clean the prefixes.

Why not fake the missing rollbacks to "complete" the folder?
An empty-looking gap that's honest beats a full-looking folder that's a lie. A placeholder rollback that doesn't actually reverse its migration would fail exactly when it's needed most — during an emergency.
