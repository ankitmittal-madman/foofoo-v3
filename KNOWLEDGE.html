Recommendation engine
The class-first RE: a household resolves to a cohort, the cohort to a weekly class plan, each class to a dish pool, and the engine picks the slate. One reusable engine owns every recommendation decision; recommendation logic exists only in supabase/functions/_shared/services/re/. Authority: DOC-P3-03 §06–11, RE-DOC-01–05, WP-8D/WP-8E.

Cold start → warm start
Day-0 confidence is computed from onboarding answers (LF-A08), clamped to the documented 0.40–0.65 Day-0 range (DCR-8E-01 cap at 0.65); cold_start_mode holds while interaction_count is low. Cold-start ranking leans on CohortPrior (w≈0.55 at tier 1) + exploration (Thompson sampling, Beta(1,1)=uniform at start, bounded bonus). As interactions accumulate, the weight ladder (5 tiers, continuous forward-transition interpolation — DCR-8D-01) shifts weight from cohort priors toward PersonalHistory/ContentMatch. Current limitation: re_cohort_class_priors is unseeded, so cold-start CohortPrior = neutral 0.50 fallback (LF-E02) until seeded.
One engine, three callers (DCR-P3-06-007; proven in the WP-8E integration test)
OnboardingOnboardingOrchestrator runs LF-A01–A09 (its own logic), then invokes engine.generateWeekPlan, persists via WeekPlanStore, returns the §06.2 handle. Per the CDM Domain Events, the RE Engine is the first-plan actor (AD-01 reconciliation, WP-8D §1).
RecommendationsRecommendationService (POST /v1/recommendations, §06.4) invokes engine.generateSlate for one slot, persists the fresh slate + FinalScores.
Nightly schedulerNightlyPlanScheduler (LF-L01, 23:30 UTC / 05:00 IST when deployed) iterates eligible users through the same engine; single-user failures isolated.
History, context & evolution
History: the engine reads the append-only event stream (never writes user tables — RE-DOC-01 isolation); PersonalHistory decays with λ=0.05; Not-Today suppression decays with threshold + context override; Never list is a hard constraint. Context: ContextFit multiplies weather/season/day against genome tags (re_context_multipliers, seeded); weather via 12h weather_cache. Evolution roadmap (RE-DOC-05, behind the stable /v1 contract via re_engine_versions): classfirst_v1 (active) → classfirst_v2 → ltr_v1 → ml_v1; /v2/recommendations reserved, unimplemented. No pgvector/LLM infra exists today by design (DOC-P4-00 §28).
