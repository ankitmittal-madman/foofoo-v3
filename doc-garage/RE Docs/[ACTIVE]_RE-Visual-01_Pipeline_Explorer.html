Layer 4 Contextual meal science outermost

Sets the frame for every request. Weather, day of week, time of day, season — all read silently before any scoring begins. A rainy Tuesday morning generates different scoring multipliers than a sunny Sunday afternoon. The user never configures this.

Weather signal
🌧 Rainy → comfort × 1.2, fried × 1.15, light × 0.8
Day signal
Tuesday (weekday) → quick-cook class bias, <30 min
Season signal
June–Sept (monsoon) → seasonal_affinity:monsoon boosted
Time signal
7 AM → breakfast slot, morning-appropriate only

Layer 3 Household food intelligence the differentiator

Resolves who this plan is for. One plan for the whole household + member-specific add-ons. Home state and current city are always separate signals — a Madhya Pradesh family in Mumbai gets MP food identity as the base, Mumbai as an overlay.

Member add-ons

Class plan

Layer 2 Food graph safety & substitution

Filters candidates before scoring. Propagates allergens through ingredient relationships — a peanut allergy automatically excludes every dish containing any peanut-derived ingredient, including dishes added to the database months later.

Allergen filter
Peanut allergy → removes groundnut oil, chikki, namkeen, satay
Diet filter
Vegetarian → removes all non-veg dishes before scoring
Class filter
BF_LIGHT_GRAIN → removes all non-breakfast-class dishes
Never list
User's Never list → permanent exclusion
↓
Layer 1 Meal genome scoring innermost · scores survivors
